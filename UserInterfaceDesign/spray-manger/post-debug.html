怎麼判斷

  • Open GET /exec：新分頁如果看到 ok:true，代表部署 URL 基本正確。
  • POST by fetch：如果失敗且出現 CORS，就是瀏覽器不允許讀 Apps Script 回應，不代表 Apps Script 沒收到。
  • POST by no-cors：前端看不到回應，狀態會是 opaque；請去 Gmail 或 Apps Script Executions 看有沒有 MAIL_SENT。
  • POST by form：用表單送 payload=JSON，會開新分頁；這通常最適合確認 Apps Script 的 doPost 是否真的被打到。
