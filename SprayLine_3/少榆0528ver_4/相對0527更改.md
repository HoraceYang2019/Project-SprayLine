# 相對0527更改

本文件說明 `少榆0528ver_4.zip` 相對於 `少榆0527ver.zip` 的主要更動。  
比較基準為本次實際檢查的兩個壓縮包：

- 舊版：`少榆0527ver.zip`
- 新版：`少榆0528ver_4.zip`

---

## 一、整體狀態

`0528ver_4` 是在 `0527ver` 的 service_extension 架構上，針對 ontology 命名、CSV 維護主檔、schema / template 對齊、threshold template、runtime template 與檢查報告進行整理後的版本。

本次版本的核心目標是：

1. 讓 Protégé / OntoGraf 顯示更乾淨。
2. 統一 line 命名規則。
3. 移除容易誤解的獨立 Controller / 控制器節點。
4. 讓 CSV templates 成為較容易維護的主檔。
5. 讓 schema / JSON template 與 CSV templates 對齊。
6. 保持 runtime、rules、output 不造假。

---

## 二、Ontology 主要更動

### 1. line 命名統一

`0527ver` 中仍可看到舊產線代號，例如：

- `M1`
- `M2`
- `M3`
- `LineM1...`
- `LineM2...`
- `LineM3...`

`0528ver_4` 已統一改為：

| 舊版概念 | 新版欄位值 | Ontology IRI local name |
|---|---|---|
| 第一條產線 | `line_1` | `Line1` |
| 第二條產線 | `line_2` | `Line2` |
| 第三條產線 | `line_3` | `Line3` |

例如：

| 0527ver | 0528ver_4 |
|---|---|
| `LineM1FilterMesh` | `Line1FilterMesh` |
| `LineM2FilterMesh` | `Line2FilterMesh` |
| `LineM3FilterMesh` | `Line3FilterMesh` |

新版 TTL 中已檢查：

- `M1 / M2 / M3`：0
- `line_id_01 / LineId01`：0
- `line_1 / line_2 / line_3`：保留作為 line_id 欄位值
- `Line1 / Line2 / Line3`：保留作為 ontology IRI local name

---

### 2. Controller / 控制器移除

`0527ver` 中 ontology 的 PhysicalAsset 相關內容仍曾出現 `Controller` 概念。  
`0528ver_4` 已調整為：

- 不再建立獨立 `Controller / 控制器` ontology 節點。
- 控制器概念若未來需要，只視為 RobotArm / 機械手臂內部控制模組，不獨立建實體。

新版 TTL 檢查結果：

- `Controller / Controler / 控制器`：0
- ontology TTL parse：全部 OK

---

### 3. 中英文 TTL 分離

`0527ver` 的主 ontology TTL 曾同時包含中英文 label，Protégé / OntoGraf 顯示時容易中英混雜。

`0528ver_4` 已整理為：

| 檔案 | 顯示語言 |
|---|---|
| `ontology/SprayLine_service_extension.ttl` | 英文顯示版 |
| `ontology/SprayLine_service_extension_chinese.ttl` | 中文顯示版 |
| `ontology/SprayLine_service_extension_english.ttl` | 英文顯示版 |

新版檢查結果：

- 中文版：`rdfs:label` 僅保留 `zh`
- 英文版：`rdfs:label` 僅保留 `en`
- 主檔：以英文顯示為主，避免混雜

## 六、Runtime / Rules / Output 狀態

### Runtime

`0528ver_4` 仍保持 runtime template，不建立假資料。

保留檔案：

- `runtime/statistics_service_output.template.ttl`
- `runtime/statistics_service_output_chinese.template.ttl`
- `runtime/statistics_service_output_english.template.ttl`

目前 runtime 仍是 template，用於未來由 service output / database output 轉成 runtime observation TTL。

---

### Rules

`rules/` 仍維持 pending。

原因：

- Past / Current 欄位仍可能依老師與負責同學討論後調整。
- Future Service 預測公式或模型尚未定義。
- threshold value / warning_rule / alarm_rule 尚未定案。
- Rule 後續需與 Database functions 對齊。

---

### Output

`output/` 仍維持 pending。

目前不建立：

- 假的 `SprayLine_runtime_observation.ttl`
- 假的 `SprayLine_runtime_inferred_sparql.ttl`
- 假的 inferred result

2. Future Service 的預測公式或模型方式。
3. threshold 實際數值。
4. warning_rule / alarm_rule 正式邏輯。
5. `sample_method`。
6. Current Service 的 `window_size`。
7. `flow_rate_ml_min` 實際單位。
8. quality / availability / utilization 的正式公式。
9. Rule 與 Database functions 的正式對接方式。
10. runtime observation / inferred output 的正式產生流程。
