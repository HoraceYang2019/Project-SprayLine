## 一、整體方向變更

原本 0527 階段主要是整理：

```text
UI / Ontology / Service / Runtime / Rules / Templates
```

後續經過多次修正後，目前已改成以 Database 同學提供的 **Dashboard v15 / DB Schema v2 / API Flow** 為主線：

```text
DB Layer
→ Service Layer
→ REST API /api/v1
→ Dashboard v15 Frontend
→ Ontology / Protégé / OntoGraf
```

舊的 UI_v4 + TimeSeriesService 中央服務方向已移除，不再作為主架構。

## 九、Protégé / OntoGraf 顯示修正

針對 Protégé 與 OntoGraf 已做多次修正：

### 1. Classes 階層修正

已補 `rdfs:subClassOf`，避免所有類別直接掛在 `owl:Thing` 下。

目前結構包含：

```text
SprayLineProjectConcept
├─ SprayLine
├─ PhysicalAsset
├─ UIComponent
├─ ServiceFunction
├─ DatabaseConcept
├─ DataResource
└─ Infrastructure
```

### 2. 實體資產階層修正

已能展開：

```text
PhysicalAsset
├─ RobotArm
├─ AirCompressor
├─ FilterMesh
├─ Nozzle
├─ SprayWidth
├─ QualityModule
└─ Oven
```

### 3. line_1 / line_2 / line_3 具體節點修正

已讓具體資產同時作為 class 與 individual，使 Protégé Classes 可展開：

```text
FilterMesh
├─ line_1／濾網
├─ line_2／濾網
└─ line_3／濾網
```

其他資產如噴嘴、空壓機、機械手臂、品質模組、烤箱、噴幅也同步處理。

### 4. 前後製程與影響方向

已補上單向流程與下游影響：

```text
AirCompressor upstreamOf Nozzle
FilterMesh upstreamOf Nozzle
Nozzle upstreamOf SprayWidth
SprayWidth upstreamOf QualityModule
QualityModule upstreamOf Oven
FilterMesh affectsDownstream Oven
Oven affectedBy FilterMesh
```

用來表達前段元件會影響後段結果，但後段不反向影響前段。
