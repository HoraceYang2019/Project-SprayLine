# 相對0528ver_4更改

本文件說明 `少榆0531ver_2.zip` 相對於 `少榆0528ver_4.zip` 的主要更動。

比較基準：

- 舊版：`少榆0528ver_4.zip`
- 新版：`少榆0531ver_2.zip`

---

## 一、版本定位

`少榆0528ver_4` 是完成 ontology、CSV templates、schema、JSON templates 與 runtime template 對齊後的穩定版。

`少榆0531ver_2` 是在 `少榆0528ver_4` 的基礎上，納入 `Project-SprayLine-main_0531` 指定內容後，再針對第三階段前發現的七個缺口進行修正的版本。

本版定位為：

> 第三階段 TimeSeriesService 整合流程驗證前置修正版。

---

## 二、本版最高命名規則

本版本所有新增與修正內容均遵守以下命名：

| 類別 | 統一命名 |
|---|---|
| 產線欄位名稱 | `line_id` |
| 產線欄位值 | `line_1`、`line_2`、`line_3` |
| Ontology IRI local name | `Line1`、`Line2`、`Line3` |
| 流量欄位 | `flow_rate_ml_min` |
| 壓力欄位 | `pressure_bar` |

已確認未使用舊產線代號、舊流量欄位、舊壓力欄位或獨立控制模組節點。

## 四、第三階段前七個缺口修正

### 缺口 1：`QueryData()` 尚未正式接 Database

本版不造假 DB output。  
改以正確 interface 檔案定義 QueryData 未來應接哪些 Database functions：

```text
docs/querydata_database_mapping.csv
```

此表明確標示：

- 查詢情境
- time_type
- database function
- input
- expected output
- used_by_service_function
- status

---

### 缺口 2：`function_io_template.csv` 不夠精準

本版新增：

```text
docs/function_service_database_mapping.csv
```

並保留 / 擴充：

```text
csv_templates/function_io_template.csv
```

重點補強：

- service function 的 input / output
- database query function
- UI_v4 region
- 是否使用 sample_method
- 是否使用 formula
- 是否需 Rule Service
- 是否仍 pending

---

### 缺口 3：`time_series_request_template.json` 缺少 `schema_version`

已修正：

```text
webservices/time_series_service/time_series_request_template.json
docs/04_WebServices_time_series_reference/time_series_request_template.json
```

新增：

```json
"schema_version": "v1.0"
```

---

### 缺口 4：JSON template 與 strict schema validation 的角色不清

本版維持 template 為 placeholder，不拿 template 當真實 sample。  
因此沒有塞假值或假 runtime data。

目前原則：

- `templates/`：看結構用，允許 placeholder。
- 真實 sample / service output：未來應由實際 service 或 DB 產生。
- runtime observation：仍 pending，不手寫假資料。

---

### 缺口 5：Past / Current 舊 reference 與新 sample_method 衝突

本版新增：

```text
docs/01_Past_Current_Service_reference/README_0531_status.md
```

並在舊 reference markdown 前方加上 0531 狀態註記。

目前採用規則：

| time_type | sample_method |
|---|---|
| past | mean |
| current | recent_average |
| future | latest_valid |

舊 reference 若有衝突，以 0531 TimeSeriesService 規則為目前版本依據。

---

### 缺口 6：公式需要的欄位來源不完整

本版新增：

```text
docs/formula_required_field_source.csv
```

此表標示每個公式欄位來源，例如：

- `flow_rate_ml_min`：DataPreprocess raw sensor / runtime metric
- `pressure_bar`：DataPreprocess raw sensor / runtime metric
- `ok_count` / `total_count`：QC result，pending
- `running_time_sec`：runtime database or production log，pending
- `planned_time_sec`：schedule or database，pending
- `rul_hr`：maintenance record or prediction service，pending
- `target_spray_width_mm`：recipe config or rule config，pending

## 六、sample_method 與公式

本版沿用 0531 TimeSeriesService 規則：

| time_type | sample_method |
|---|---|
| past | mean |
| current | recent_average |
| future | latest_valid |

公式記錄於：

```text
knowledge/time_series_sample_method_formula.template.json
knowledge/time_series_sample_method_formula_notes.md

```

注意：  
公式是 TimeSeriesService 的數值計算規則。  
Rule Service、threshold、warning / alarm、risk_text、fault_detail 仍未正式定案。

## 七、仍維持 pending 的部分

本版沒有產生假資料。以下仍 pending：

1. QueryData 的正式 DB 連線。
2. 真實 Database output。
3. 真實 UI_v4 service 串接。
4. Rule Service 正式規則。
5. threshold 實際數值。
6. warning / alarm 判斷邏輯。
7. Future Service 模型與預測資料來源。
8. runtime observation TTL。
9. inferred output TTL。
10. `component_overview`、`fault_detail`、`risk_text` 的正式產生規則。
