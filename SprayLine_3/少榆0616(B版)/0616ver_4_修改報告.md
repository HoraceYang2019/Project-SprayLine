# 0616ver_4 修改報告

## 版本定位

`少榆0616ver_4` 是在 `少榆0616ver_3` 基礎上補強 DB 實測與 Timer 重複告警問題的版本。

本版對照 GitHub `Project-SprayLine-main0616_2000pm` 後確認：

```text
Database/versionB 已提供少榆端需要的 db_sensor / db_alert / db_status / db_future function。
0616 20:00 新增的 WebServices/time_series_service_B 屬於另一條 UI / TimeSeries demo service 線，少榆正式線不改走 HTTP endpoint。
```

## 本版新增 / 修改

### 1. duplicate alert suppression

新增：

```text
webservices/monitoring_worker/duplicate_alert_guard.py
```

修改：

```text
webservices/monitoring_worker/alert_event_writer.py
webservices/monitoring_worker/config.py
webservices/integration_adapter/database_versionb_adapter.py
```

功能：

```text
同一 batch_id + station_id + sensor_name + state + cause_id
在 SPRAYLINE_DUPLICATE_ALERT_SUPPRESSION_MINUTES 分鐘內若已有未確認 alert，
則不重複寫入 alert_event。
```

預設：

```text
SPRAYLINE_DUPLICATE_ALERT_SUPPRESSION_MINUTES=5
```

### 2. DB 實測腳本

新增：

```text
scripts/check_db_connection.py
```

```text

run_db_smoke_test.py：可只讀，也可用 --write-test-data 寫入一筆測試異常資料並跑 Monitoring / Future。
```


```bash
python scripts/check_db_connection.py
python scripts/run_db_smoke_test.py --write-test-data --station Station_1
```
