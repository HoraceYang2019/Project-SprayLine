# 使用說明與開啟檔案執行流程

## 一、版本定位

`少榆0616ver_4` 是以 GitHub `Project-SprayLine-main_0616_1400pm` 為 base 的少榆端整合版本。  
本版不做正式 API Server，不走 HTTP 對外路由，而是依余宇承 0615 回覆方向：

```text
直接 import Database/versionB 的 Python function 使用
```

## 三、設定 Database/versionB import path

本版 adapter 會依序尋找：

```text
1. SPRAYLINE_DB_FUNCTION_PATH
2. SPRAYLINE_PROJECT_ROOT/Database/versionB
3. 本包 external/Database/versionB
4. 從目前檔案往上找 Database/versionB
```

若使用本包內附 reference copy，可以不用設定。若要使用 GitHub 最新版，建議設定：

PowerShell：

```powershell
$env:SPRAYLINE_PROJECT_ROOT="C:\path\to\Project-SprayLine-main"
```

或直接指定 versionB：

```powershell
$env:SPRAYLINE_DB_FUNCTION_PATH="C:\path\to\Project-SprayLine-main\Database\versionB"
```

## 四、設定 PostgreSQL 連線

余宇承 `Database/versionB/db_connection.py` 使用以下環境變數：

```text
DB_HOST
DB_PORT
DB_USER
DB_PASSWORD
DB_NAME
```

PowerShell 設定範例：

```powershell
$env:DB_HOST="localhost"
$env:DB_PORT="5432"
$env:DB_USER="postgres"
$env:DB_PASSWORD="你的密碼"
$env:DB_NAME="sprayline"
```

Windows CMD 設定範例：

```cmd
set DB_HOST=localhost
set DB_PORT=5432
set DB_USER=postgres
set DB_PASSWORD=你的密碼
set DB_NAME=sprayline
```

Linux / macOS：

```bash
export DB_HOST=localhost
export DB_PORT=5432
export DB_USER=postgres
export DB_PASSWORD=你的密碼
export DB_NAME=sprayline
```

## 五、執行順序

### 1. 安裝套件

```bash
pip install -r webservices/requirements.txt
```

### 2. 檢查 Database/versionB adapter

```bash
python -m webservices.integration_adapter.database_versionb_adapter
```

應該會顯示目前使用哪一個 `Database/versionB` 路徑，以及：

```text
integration_mode = direct_python_import
http_endpoint_used = false
```

### 3. 執行 Future payload demo

```bash
python -m webservices.future_service.future_service
```

此步驟只產生 payload，不寫 DB，因此不需要 PostgreSQL 連線。

### 4. 執行 MonitoringWorker

```bash
python -m webservices.monitoring_worker.monitoring_worker
```

此步驟會透過 `Database/versionB` function 查：

```text
sensor_1min
sensor_3min
```

並在偵測到 warning / fault 後呼叫：

```text
db_alert.insert_alert_event
db_alert.link_alert_cause
db_alert.link_alert_response
db_status.get_batch_station_status
db_status.upsert_batch_station_status
```

因此此步驟需要 PostgreSQL DB 已啟動，且 `setup_db.sql` 已建立資料表。

## 六、目前工作流

```text
DataPreprocess
-> Database/versionB 寫入 sensor_1min / sensor_3min
-> 少榆 MonitoringWorker import db_sensor 查資料
-> 少榆 EventRule 用 rules/sensor_thresholds.json 判斷 warning / fault
-> 少榆 detection_mapping 補 issue_state / cause_id / response_id
-> 少榆呼叫 db_alert.insert_alert_event 寫入 alert_event
-> 少榆呼叫 db_alert.link_alert_cause / link_alert_response 建立關聯
-> 少榆呼叫 db_status.get_batch_station_status 讀取目前快照
-> 少榆呼叫 db_status.upsert_batch_station_status 更新完整狀態快照
-> 少榆 FutureService 產生 future_prediction_result
-> 少榆呼叫 db_future.insert_future_prediction_result 寫回 DB
-> Manager UI / Engineer UI 顯示
```
