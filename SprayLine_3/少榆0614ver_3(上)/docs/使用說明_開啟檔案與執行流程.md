# 使用說明與開啟檔案執行流程

## 一、這包檔案的定位

本資料夾是少榆端的 0614ver_3 整合版本。  
本版不提供正式 API Server，正式 Database / API endpoint 由余宇承端提供。

少榆端負責：

```text
Future 預測 Service
Timer / Monitoring Worker
EventRule 判斷
Alert / Prediction payload
Troubleshooting 原因對策
Ontology / Knowledge 對應
與 DB/API 端的資料需求整理
```

不負責：

```text
正式 FastAPI Server
正式 DB insert / query endpoint
Manager UI / Engineer UI 前端實作
```

## 三、執行流程

### 1. 安裝套件

```bash
cd 少榆0614ver_3
pip install -r webservices/requirements.txt
```

目前需求：

```text
psycopg2-binary
pandas
```

本版已移除 FastAPI / uvicorn，因為正式 API Server 不屬於少榆主責。

### 2. 執行 Future payload demo

```bash
python -m webservices.future_service.future_service
```

作用：

```text
產生一筆 future_prediction_result payload
計算 risk_level
不寫入 DB
等余宇承補上 future_prediction_result table / API 後再接正式寫回
```

### 3. 執行 Monitoring Worker 一次

```bash
python -m webservices.monitoring_worker.monitoring_worker
```

作用：

```text
讀 sensor_1min / sensor_3min
依 threshold 判斷 state
產生 alert_event
更新 batch_station_status
```

注意：這個步驟需要本機或整合測試 DB 連線。  
正式流程會改成呼叫余宇承提供的 DB/API。

### 4. 本機 DB 測試用環境變數

```bash
set SPRAYLINE_DB_HOST=localhost
set SPRAYLINE_DB_PORT=5432
set SPRAYLINE_DB_NAME=sprayline
set SPRAYLINE_DB_USER=postgres
set SPRAYLINE_DB_PASSWORD=postgres
```

Linux / macOS 可改用：

```bash
export SPRAYLINE_DB_HOST=localhost
export SPRAYLINE_DB_PORT=5432
export SPRAYLINE_DB_NAME=sprayline
export SPRAYLINE_DB_USER=postgres
export SPRAYLINE_DB_PASSWORD=postgres
```

## 四、端到端示範目標

老師下次最可能想看的是這條流程：

```text
林聖堯產生異常資料
→ 余宇承 DB/API 寫入 sensor_1min / sensor_3min
→ 少榆 MonitoringWorker 讀到資料
→ threshold_evaluator 判斷 state
→ 產生 alert_event
→ 透過余宇承 DB/API 回寫 alert_event
→ Engineer UI 顯示完整原因對策
→ Manager UI 顯示摘要與風險狀態
```

Future 預測流程則是：

```text
sensor_1min / sensor_3min / batch 資料
→ FutureService
→ future_prediction_result payload
→ 透過余宇承 DB/API 寫入 future_prediction_result
→ Manager UI 顯示 predicted_ok_rate / predicted_ng_count / risk_level
```
