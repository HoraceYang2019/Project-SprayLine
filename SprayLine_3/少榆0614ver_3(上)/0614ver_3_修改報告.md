# 0614ver_3 修改報告

## 修改背景

`0614ver_2` 中包含正式 `webservices/api_server.py` 與 FastAPI 啟動方式，容易讓老師誤解少榆需要負責完整 API Server。  
依重新判斷逐字稿與分工，正式 API / DB function 應由余宇承端提供，少榆端主責是 Future Prediction Service、Monitoring / Timer、EventRule、Troubleshooting 與 Ontology / Knowledge 對應。

## 本版主要修改

### 1. 刪除正式 API Server

已刪除：

- `webservices/api_server.py`
- `webservices/time_series_service/`
- `webservices/batch_service/`
- `webservices/dashboard_service/`
- `docs/contracts/api_routes_0614.csv`

這些內容容易被誤解為少榆負責正式 API 後端，因此移除或改成需求文件。

### 2. 新增余宇承 DB/API 需求文件

新增：

- `docs/contracts/api_requirements_for_yucheng_0614.csv`
- `docs/contracts/shaoyu_service_scope_0614.csv`

用來明確標示少榆需要余宇承提供哪些 DB/API，例如：

- 查 `sensor_1min`
- 查 `sensor_3min`
- 寫入 `alert_event`
- 更新 `batch_station_status`
- 新增並寫入 `future_prediction_result`

### 3. 保留少榆主責 service logic

保留並調整：

- `webservices/future_service/future_service.py`
- `webservices/monitoring_worker/`
- `webservices/event_rule_service/`
- `webservices/troubleshooting_service/`

這些檔案定位為少榆端 service logic，不是正式 API Server。

### 4. 調整 WebServices 說明

已更新：

- `webservices/README.md`
- `README.md`
- `docs/notes/0614_integration_action_plan.md`

內容明確說明：

```text
少榆不負責正式 API Server。
正式 DB/API 由余宇承提供。
少榆負責 Future / Monitoring / EventRule / Troubleshooting 邏輯與 payload。
```

### 5. 新增使用說明與開啟檔案執行流程

新增：

- `docs/使用說明_開啟檔案與執行流程.md`

內容包含：

- 建議開檔順序
- 少榆主責與非主責範圍
- Future demo 執行方式
- Monitoring Worker 執行方式
- 與余宇承 DB/API 端的串接流程
- 下次展示目標

### 6. 調整 Ontology service module

`ontology/service/SprayLine_service_ontology.ttl` 已移除正式 RESTEndpoint 的呈現方式，改為：

```text
ShaoYuService
ExternalAPIRequirement
ServiceFunction
```

用來表示少榆端服務邏輯與外部 DB/API 需求，而不是表示少榆已完成正式 API Server。

### 7. requirements 修正

已移除：

- `fastapi`
- `uvicorn`

保留：

- `psycopg2-binary`
- `pandas`

## 本版定位

```text
少榆端 service logic + integration contract
```

而不是：

```text
少榆端完整 backend API server
```
